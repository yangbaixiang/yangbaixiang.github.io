Hash函数

定义：hash函数即散列函数，可以将不定长的输入，通过散列算法得到定长的输出，不同的输入值可能会得到相同的输出值，所以散列函数并不能去的唯一的值

常见的散列函数算法

1.直接定址法

直接取关键字K或者k加上某个常数(K+C)作为hash地址
Ｈ(key) = key 或 H(key) = a * key + b
其中K，C为常数，这种定址方法所得的地址的集合和关键字的集合大小相等，所以对于不同的关键字所得的地址不会出现冲突的现象，不是实际使用这种方式较少。

2.数字分析法

提取关键字中取之比较平均的数字作为hash地址

数字分析法的基本要求
1.关键字是以R为基数(如已知关键字都是10进制，或者8进制)
2.哈希表中可能出现的关键字是已知的。
对所有的关键字进行分析，查找相似度最低的某位或者某几个的数字，然后根据每个关键字上的这几位进行求哈希值

3.分段叠加法

按照哈希地址位数将关键字分为位数相等的几个部分（最后一部分的位数可以不同），然后将这几个部分叠加，舍弃最高进位就是该关键字的哈希
地址，适用于关键字很多，且关键字中的每一位上的数字分布大致均匀

例如对0-442-20586-4进行分段叠加取哈希值

4.平方取中法

取关键字的平方之后的中间几位为哈希地址，一个数平方之后的中间几位和这个数的每一位都有关系，由此使用随机分布的关键字得到的哈希地址也是随机的，取的位数由表长决定

5.伪随机数

选择一随机函数，取关键字作为散列地址,关键字的长度不等时，可以采用此方法

6.除留取余法

取关键字被某个不大于哈希表表长m的数p除后得到余数为哈希地址
H(key) = key MOD p , p<=m
这种方式不仅可以对关键字直接取模，这可以在折叠，平方取中的基础上取模，p的取值需要技巧，如hashMap 取p为数组长度-1的值

选取哈希函数

选取哈希函数需要考虑的因素

  1. 计算哈希函数所需要的时间
  2. 关键字的长度
  3. 哈希表的大小
  4. 关键字的分布
  5. 记录查找的频率

哈希冲突的解决办法

1.链地址法

将哈希表的每个单元作为链表的头节点，所有哈希地址为i（即哈希地址值相同的一类）的元素构成一个同义词链表，即发生冲突就把该关键字链在以该单元为头节点的链表尾部

2.开放定址法

发生冲突时，寻找下一空的哈希地址值，只要哈希表足够大，总能找到空的哈希地址

3.再哈希法

发生冲突时，再由其他的哈希函数重新计算一次哈希值

4.建立公共益处区

将哈希表分为基本表和益处表，发生冲突时将冲突的元素放在益处表

key的hash值与数组长度-1做”与”运算得到key所在数组tab上的索引位置

如图左边两个数组的长度为16(2的4次方)，右边的两个数组的长度为15，分别对这四组数组长度值-1值与8和9做”与”运算(可以理解为分别用15和14 对 8和9做与运算 )

根据结果可以得出当8和9对14的二进制值做”与”运算的时候得到了相同的结果1000，也就是说如果在HashMap中8和9会被放到相同的索引下，那么查询的时候就会降低效率。

同时根据分析可以得出当数组的长度为15的时候，即hashcode的值会与14(1110)做”与”运算，那么得到的结果最后一位永远是0，像0001，0011，0101，1001，1011，0111，1101这几个位置永远不能存放元素，造成数组空间浪费，同时数组中可使用的位置比数组的长度小了很多，进一步加大的碰撞的概率

所以当数组长度为2的n次幂的时候，不同的key得到的index相同的几率较小，数据在数组上分布均匀
HashMap数组的默认长度为16也正是因为这个
在手动指定HashMap数组长度的时候最好也是用2的n次幂

HashMap数组的扩容

HashMap扩容的时间，当HashMap中的元素个数超过数组大小loadFactor时便会自动扩容(loadFactor的默认值为0.75),每次扩容容量扩大一倍
扩容时HashMap对数组中所有的元素重新做了一次位置的计算，这个最消耗内存的操作
因此在如果能已知HashMap所需要存储元素数量的情况下，最好手动指定数组大小，指定数组的大小0.75最好大于已知元素的数量(在指定数组大小为2的n次幂的前提下)

Handle原理

三个重要的类 ：Handle，Looper，MessageQueue
Handle：封装了消息的发送，消息的接受，内部与Looper关联

ThreadLocal实现Looper线程唯一的原理

ThreadLocal：实现了线程本地的存储， 所以的线程共享一个ThreadLocal对象，但是不同的线程仅仅能访问与其值关联的值，一个线程修改了ThreadLocal对象，其他的线程不受影响

ThreadLocal可以理解为一块存储区，将一块大的存储区分割成若干个小的存储区，每一个线程都拥有一块属于自己的存储区，那么每个线程操作自己的存储区就不会影响其他的线程，在消息机制中，每一个小的存储区就保证了每个线程和当前线程的Looper想关联

ThreadLocal的实现原理

Thread的成员变量localValues代表了线程特定的变量，类型为ThreadLocal.Values，由于线程特定的变量可能会有多个，并且类型不确定，所以ThreadLocal.Values有一个table成员变量，类型为Object数组，LocalValues可以理解为二维存储区中与特定的线程相关的一员。
ThreadLocal类相当于一个代理，真正操作线程特定存储区的是其内部类Values

流的分类

输出流：OutputStream / Writer

按流的功能分：节点流和装饰流(处理流)

装饰流用来封装节点流,从而给节点流增加一个功能,不能独立存在,在关闭流时如果使用了装饰流,只需关闭最外层的流就可以了。

    1. 字节流：为了处理字节而生。 读取图片、视频、音乐等数据时使用。
    2. 字节流可以读取各种数据类型。
    3. 字符流：为了处理字符。读取文字、符号等文本时使用。
    4. 字符流,起步就是一大串子的字符,绝对不会去使用字符流去读取某一数据类型的。

  5. 在序列化对象的时候，应该为该类创建一个serialVersionUID属性，做为字节码的ID，若是不定义此变量则系统会依据该类内的属性名、属性的标识符等自动生成的一个serialVersionUID，因此若是以后该类的某个字段的名字被修改了，则自动生成的serialVersionUID就不一样了。

基础知识

程序，进程，线程

程序是一段静态的代码，他不占内存空间，不受操作系统的调度，不能作为独立的运行单元，也不占用系统的运行资源，只占硬盘空间

进程是程序的一次动态的执行，占据内存空间，受操作系统的支配，可以作为独立的运行单元
操作系统按照同时可以执行的进行数分为单进程和多进程
Dos是单进程的，整个系统只能有一个进程在执行
Windows是多进程的
进程的建立至少需要一个线程：主线程做为程序的入口

线程是程序运行的基本单元，线程隶属于进程

  1. 一个进程可以用多个线程，当一个进程消失，那么进程中的所有线程全部销往
  2. 一个线程死亡了进程可能仍在运行，线程不是程序不能单独运行
  3. 多线程 简单的说就是可以使一个进程在同一个时间段内做多个工作

并发和并行

并发：多个线程轮流执行，由于线程间的切换的速度非常快，所以感觉上是同时执行，如多线程同时下载文件
线程不是越多越好，线程多不但不会提高效率，反而会增加CPU在多个线程间切换的时间

并行：只有在计算机有多个CPU(或一个CPU多个核心)时，线程才能真正的同时执行

当程序中有多条执行路径并发执行时，就称为多线程
所有的线程共享进行最初从内存中申请的空间，当新建一个线程时，是在进程所占的内存为其分配空间，而不是在内存的其他地方重新申请

时间片轮询

多线程下实行CPU时间片的轮询
每个进程被分配一个时间段，称作他的时间片，即该进程允许运行的时间，如果在时间片结束时进程还在运行，则CPU将被剥夺并且分配给另一个进程，如果进程在时间片结束前阻塞或结束，则CPU当即进行切换。在进程的多线程中同样采用时间片轮询

每个线程通过优先级来确定谁先占用CPU
即使在主线程中启动了其他的线程，但是在主线程的时间片没结束之前，那些线程仍不能被执行，他们现在处于就绪状态，除非在就绪队列中，某个线程的优先级比当前正在执行的线程的优先级高
java执行main方法中的线程就是主线程

java中的线程

当一个java程序启动的时候，JVM会创建主线程，并且在该线程中调用程序的main方法

  1. 主线程是程序最早执行的线程，程序中所有的线程都是由主线程创建的
  2. 程序在启动的时候，JVM还创建了其他的线程。如垃圾回收gc线程，以及对象终止有关的其他线程
  3. java中多线程抢占的方式：若当前就绪的线程中优先级高于正在执行的线程，则CPU会放弃当前的线程的执行， 去执行优先级高的线程，即使当前执行的线程是主线程也不例外

注：进程之间各自在自己的内存空间中独立运行，进程之间不共享内存
进程内所有的线程共享进程空间，线程也可以拥有自己的内存空间，这个空间叫做线程栈，主要用来保存线程内部所使用的数据
OS为了提高程序的运行效率，在一个线程空闲的时候，会撤下整个线程，让其他的线程来执行，这种方法叫线程的调度

线程的状态

java中线程一共分为5

种状态

  1. 新建状态
    使用Thread t = new Thread() 语句建立一个线程对象，新的线程就处于就绪状态了
    此时它只是一个空的线程对象。若要它执行，还需要将这个线程在OS中进行登记并为它分配系统资源，这是工作通过start()方法来完成的
  2. 就绪状态
    当一个线程调用了start()方法之后，它便进入了就绪状态，当时并不会马山执行，他们会被放到就绪队列中等待执行，何时执行取决于线程的优先级以及就绪队列的当前状况
  3. 执行状态，线程被分配到CPU时间片，开始运行run()方法，此时线程处于执行状态
  4. 阻塞状态
    因为一些原因导致线程需要等待，此时处于等待状态的线程就称为阻塞
    常见的阻塞的原因:1. 线程自己调用了 sleep()方法 2.线程自己调用了wait()方法 3.线程等待用户的输出或者输出操作
    相应的解决办法 ：1. 等待睡眠时间结束，或其他的线程调用interrupt()方法将其唤醒 2. 等待它等待的时间结束，或其他的线程调用notify(),notfyAll()方法将其唤醒 3.等待用户输入或输出结束
  5. 死亡状态
    一个线程被创建之后只能调用一次start()方法，当线程完成其run()方法时，该线程不再是一个可执行的线程，该线程的栈已经解散，我们就认为该线程死亡，永远无法被重启，
    造成该线程死亡的三种情况 ：1. 正常的执行run()方法 2.在run()方法中因为一个未捕获的异常终止run()方法，导致线程猝死 3.是用stop()或者是destory()方法终止线程，注意：stop会产生异常，dertory是强行终止吗不会释放锁

协作式多任务：协作式多任务调度是在一些小型的设备中使用，在这样的设备中，一个线程只能通过sleep或yield方法，才能释放控制权，协作式多任务下，一个行为不当的程序可能独占资源，导致其他的线程无法继续工作

正则表达式的概念

在计算机科学中，正则表达式是指一个用来描述或者匹配一系列符合某个句法规则的字符串的单个字符串。

正则表达式和普通表达式的相同点

  1. 正则表达式本质上也是一个字符串。如,字符串“a*”就是一个正则表达式。
  2. 正则表达式也是由若干个有特殊意义的符号组成的表达式。
  3. 普通表达式是由数字、算符、数字分组符号组成，正则表达式也是，如“*^+?”等。
  4. 两者都是会去有意义的排列符号(不会去胡乱排列)。比如表达式“4-3=1” 不会被写成 “-43=1”，正则式“a”不会写成“a”

正则表达式和普通表达式的不同点 ： 普通表达式以能求得数值为目标。 正则表达式以匹配字符串为目标

面向对象和过程

  1. 面向过程设计中 专注于解决问题 导致设计不合理 代码的间联系紧密 因此不利于代码维护(代码修改)。

  3. 与面向过程相反。 在面向对象设计中 代码易维护、可重用。
  4. 面向过程编程 还有一个特点：代码自顶向下 一步一步 顺序执行。(即 这一步做完后才进行下一步操作)。
  5. 面向过程是面向对象的基础。 不管你怎么分析问题 一切的分析都结束后 仍然需要按照面向过程的编程思路 “自顶向下 一步一步 顺序执行”进行具体的编码。
  6. 在大的方向上使用面向对象分析 在具体的实施阶段上 使用面向过程编程。

  • 对象a改变这个变量的值 对象b再引用此变量的时候 引用的值 是已经改变之后的了。

内存

操作系统中内存的划分

  1. 静态数据区：存放程序中的静态数据，如static变量、常量等。变量的内存地址在程序编译的时候就已经分配好，这块内存在程序的整个运行期间都存在。
  2. 代码区：存放程序中源代码。如函数、语句等。
  3. 栈：存放简单变量、对象的引用变量。当程序流程走出局部变量所在的代码块后，局部变量会被从栈中弹出。其所占内存会被操作系统自动回收。
  4. 堆：存放程序动态申请的数据。如Java中的对象(的实例属性)和C语言使用malloc()函数申请的空间都保存在堆中。 保存在堆中的数据，由于是用户自己动态申请的空间，所以当堆中的数据不再需要被使用时，则应该由用户手工释放该数

堆和栈

在操作系统中，栈和堆的容量都是有限的。

  • 栈是内存中的一片连续的存储空间，类似于数组，每当需要为函数中局部变量开辟空间时，都会从栈中申请。
  • 若栈中的内存足够，则系统将为程序提供内存，否则将报异常并提示栈溢出。
  • 当函数执行完毕后，会按照后进先出的原则，将局部变量依次弹栈，并将释放出的资源加入到栈中的空闲内存中。
  • 堆是由内存中的多个分散的存储空间组成的一个不连续的存储空间，各个存储空间基于数据结构中的链表，而组织在一起。
  • 当系统收到程序的空间申请时,会遍历该链表,寻找第一个空间大于所申请空间的堆结点,然后将该结点从空闲结点链表中删除,并将该结点的空间分配给程序。

堆和栈的优缺点

栈: 由系统自动给变量分配空间，虽然分配空间的速度较快，但是容量固定。
堆：空间的大小由用户自定义，分配空间的速度一般比较慢，而且频繁的申请、释放空间容易产生内存碎片,不过用起来最方便。

内存

内存溢出：通俗的说，就是当前操作系统中的剩余内存容量不足。
内存泄漏：通俗的说，堆中存在很多已经无用的空间，这些空间在被使用过后，没有被程序员释放，导致系统中明明存在可用的内存空间，却偏偏不能使用。
栈溢出：栈中剩余空闲空间不足以满足应用程序所需要的空间

方法重写

定义 ： 在子类中定义一个与父类方法中同名的方法

方法重写的要求

  1. 子类中重写的方法的访问权限不能小于父类中方法的访问权限。可以增大访问权限。

    • 子类方法的返回值，必须和父类方法的返回值相同。 如果父类返回double型 则子类也必须是double。
    • 或者子类方法返回值的类型是父类方法返回值类型的子类。
    • 子类中的方法必须和父类中的方法同名。
    • 子类方法的参数列表必须和父类方法的完全相同

  2. 参数。列表的内容包括：参数的个数、顺序、类型。

  3. 如果父类方法抛出了异常

  • 子类方法可以不抛异常。
  • 子类方法可以抛出父类方法抛出的异常。
  • 子类方法不可以抛出父类方法没有抛出的异常。
  • 子类方法抛出的异常可以是父类方法抛出的异常的子类。 如果父类方法抛出的异常是Exception 则子类方法可以抛出任意类型的异常。
  1. 父类中的private、final修饰的方法不会被重写。
  • 子类无法继承父类中的使用private。如果你重写成功了，那只不过是重新定义。
  • final修饰的方法 被称为最终方法 不能被重写。即“可以被继承 但不能重写”。
  • Static修饰的方法可以被重写、继承

由于在子类中定义了一个与父类中同名的实例变量。 因此 父类中的变量被隐藏起来了。 不再起作用。

  2. 如果希望调用父类中被隐藏的属性 可以在子类中使用super关键字。
  3. 子类中的属性可以和父类中的属性的类型不相同 。只要变量名相同 就可以达到属性隐藏的效果。
  4. 实际开发中 属性都是private的 因此 关于属性的隐藏了解即可

特点：

  1. 普通的外部类只用public和default访问权限。内部类还增有protected、private。
  2. private内部类只有其外部类可以访问。
  3. protected内部类只有其同一个包中的类和不同包中的其外部类的子类可以访问。
  4. 内部类的全名： 外部类名.内部类名。
  5. 外部类和内部类之间可以互相访问对方的private变量。
  6. 内部类生成的.class文件 命名为：外部类名$内部类名.class

异常处理

定义：异常处理机制是指当程序出现错误后，程序如何处理。具体来说，异常机制提供了程序退出的安全通道。当出现错误后，程序执行的流程发生改变，程序的控制权转移到异常处理器

异常类的结构

Object 类派生出Throwable类,Throwable类是Java中最大的处理异常的类。其又有2个子类

1.Error类
Error一般表示由程序运行异常或其他原因导致的一个JVM错误。它们通常是一个致命的错误，不是一般程序能控制的。因此 在程序中 一般也无法解决此类问题。例如： 使用java命令执行一个不存在的类就会抛出一个：” Exception in thread “main” java.lang.NoClassDefFoundError”

2.Exception类一般是指程序中可以进行处理的错误 如：除数为0、数组下表越界等。其本身又分为两类

  1. RuntimeException 异常：此类表示运行时的异常。即在程序运行的时候，由于某些变量的值非法等导致的错误。若出现RuntimeException，那么一定是程序员的错误
  2. 其他非RuntimeException异常，如IOException 。这类异常一般是外部错误，并不是程序本身的错误，而是在应用环境中出现的外部错误

异常处理的方式

使用try…catch块来捕获异常。
try…catch块之间的语句代表是可能产生异常的语句。
一个语句可能会产生多种异常(但是在某一瞬间上 只会产生一种异常) 。此时可以使用多个catch语句。
finally块中的语句不论程序中是否抛出异常 都会去执行

异常匹配的原则

  1. catch块中的引用变量与产生的异常对象属于同一个类。
    |- 如抛出了ArithmeticException异常 而某一个catch块后恰巧有这个类的引用变量。
  2. catch块中的引用变量的类型是异常对象的父类。
    |- 如抛出了ArithmeticException异常 而某一个catch块后有一个Exception类型的引用变量。Exception就是ArithmeticException异常的父类。
  3. catch块中的变量是一个接口类型的引用变量时 产生的异常对象实现了此接口。
  1. 在方法头部写上throws关键字 代表这个方法可能抛出异常。
  2. throws关键字后面列出的是本方法可能抛出异常列表。如果本方法可能抛出多个异常则多个异常之间使用逗号间隔。
  3. 由于使用了throws关键字 就代表此方法可能抛Exception类型的异常。因此在方法的调用处必须对这些异常进行处理。 在main方法中也可以使用throws关键字 此时会将异常抛给JVM处理。
  4. 当异常产生时 程序会立刻终止当前行代码的执行 然后跳到catch块中匹配。当异常处理完成后 程序不会回到异常产生的位置继续执行 而是从异常处理语句之后的第一条语句开始执行

注意 ：
程序中RuntimeException类和其子类抛出的异常称为运行异常。
在方法定义的异常清单(即 throws 后面的列表)中没必要列出运行异常。
try…catch块可以嵌套 使用。 如果在一个内部try中产生了异常 且在其子catch中并无匹配项 则会跳出内层前往外层catch中进行匹配 如果仍未匹配成功 则异常将抛给JVM
